Architecture at a glance

The app follows a one-way data flow: the backend supplies choices, the container owns user selections, a composable derives a result, and presentation components render it.

Container layer

LoanCalculator.vue is the coordinator. It creates the four pieces of form state, converts select values to numbers, finds the selected annual rate, and connects the composables to the child components.

It does not contain the finance formula, which keeps UI concerns separate from business logic.

Domain layer

calculateRepayments.js is a pure function. Given four numbers, it validates them, calls the supplied PMT utility, fixes the outflow sign, rounds money, and returns the result.

Because it has no Vue or network dependency, it is easy to test independently.

Core design principle

Data moves downward as props and computed values. User changes move upward through v-model. The calculation remains isolated from rendering.

Branching states

Loading

isLoading = true; selects are disabled and the loading message appears.

Incomplete or invalid

The computed result is null; a placeholder replaces the summary.

Fetch error

Lists are cleared, selections are reset by the watcher, and retry is offered.

Why the error watcher clears selections

A later failed reload should not leave an old annual rate or period selected. Resetting the selects also makes the computed result become null, preventing a stale repayment estimate.

Why each transformation exists

Divide annual rate
PMT needs the interest rate for one repayment period, not one year.
Convert term to periods
A two-year weekly loan has 104 payments; monthly has 24.
Use Math.abs
PMT returns cash paid out as a negative value; the UI displays a positive cost.
Round to cents
Displayed and multiplied monetary values remain aligned to currency precision.
Interview nuance: total repayment rounding

This implementation multiplies the rounded periodic payment by the period count. That models paying the displayed cent amount each period. An alternative is multiplying the unrounded PMT value and rounding only the final total; explain which business rule you chose.

A concise architecture answer

“The container owns state. One composable fetches API options, another derives the reactive result, and a pure utility owns the finance formula. Small components handle input and presentation. Unit tests protect the math; Cypress protects the complete user flow.”
